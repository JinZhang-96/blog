ThreadLocal 浅析

发表于 2023-09-05 更新于 2023-09-06 分类于 Java ， ThreadLocal 阅读次数： 本文字数： 2k 阅读时长 ≈ 7 分钟

ThreadLocal 是位于 java.lang 包下的类，主要的作用是存储线程的私有数据。
这些数据最终会存放到 java.lang.Thread 类的 threadLocals 变量内。

准备

学习 java.lang.ThreadLocal 类，要用到访问修饰符相关知识，先回顾一下 Java 中的每种访问修饰符可访问的范围。

API

ThreadLocal 可用的 API 比较简单，共有 4 个方法：

方法名 类型 作用 示例
withInitial 静态方法 设置 ThreadLocal 的初始数据 ThreadLocal<String> threadLocal = ThreadLocal.withInitial(() -> "初始数据");
get 类方法 获取数据 threadLocal.get()
set 类方法 设置数据 threadLocal.set("设置数据")
remove 类方法 删除数据 threadLocal.remove()

如果 threadLocals 为空，则通过 ThreadLocal createMap() 方法为 threadLocals 赋值；否则， 以当前 ThreadLocal 的实例为 key, 调用 ThreadLocalMap set(key, value) 方法设置数据。

get

具体调用过程如下：

分析 ThreadLocal 源码，可知调用 ThreadLocal get()，是先获取当前线程的 threadLocals 变量。

如果 threadLocals 为空，则调用 setInitialValue() 方法，其方法会通过 initialValue() 方法获取初始数据，然后通过 ThreadLocal createMap() 方法为 threadLocals 赋值，并返回初始值。

否则， 以当前 ThreadLocal 的实例为 key, 调用 ThreadLocalMap map.getEntry(this) 方法获取一个 ThreadLocalMap.Entry 实例， 然后返回ThreadLocalMap.Entry 的 value 属性。

Entry 类继承了 WeakReference 类，这导致 Entry 类持有的 ThreadLocal 实例是一个弱引用， 垃圾回收的时候，这里的引用不能阻止 ThreadLocal 实例被回收。 一旦 key 被回收后，这个 Entry 实例就再也不能被找到， value 就永远的留在了 Entry 的实例中。 这将有内存泄漏的风险。
所以在使用 ThreadLocal 的时候， 如果 ThreadLocal 的实例不用了，需要 调用 remove() 把数据清除掉， 防止内存泄漏。

思考

既然 ThreadLocal 实际是操作的 Thread threadLocals 变量， 为什么不使用 public 修饰变量， 而用 default 修饰变量呢？
public 修饰变量， 则可以 随意的设置 key；而 default 修饰变量, 只能同一包的类访问这个变量。
这就和 TreadLocal 绑定在一起了， key 值只能采用 ThreadLocal threadLocalHashCode。
